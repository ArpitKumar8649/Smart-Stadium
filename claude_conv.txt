  ---
  Concourse (Smart-Stadium) — Full Project Analysis

  What this is

  A context-aware smart-stadium companion built for PromptWars Virtual Challenge
  4 (FIFA World Cup 2026). A fan-facing React PWA backed by a
  TypeScript/Express API where a Qwen LLM concierge is force-grounded in
  deterministic tools (venue lookup, A* indoor routing, crowd state, optional
  Google outdoor routes). Live at concourse-stadium.web.app (Firebase) + Azure
  API.

  It's an npm workspaces monorepo (shared, backend, frontend), Node 22, strict
  TS, Vitest + Playwright, GitHub Actions CI + dual deploy (Azure App Service
  backend, Firebase Hosting frontend).

  frontend/  React 18 + Vite PWA (Cesium 3D + Leaflet 2D + SSE chat)
  backend/   Express + Qwen agent + A* router + crowd sim + ASR/TTS/vision
  shared/    Zod schemas (single source of truth for contracts)
  data/      MetLife venue graph (3,479 nodes / 8,167 edges) + fixtures + system
  prompt
  stadium/   Mappedin-exported GeoJSON/JSON (source for frontend/public)

  pino (with redaction), Zod, openai SDK pointed at DashScope's
  OpenAI-compatible endpoint, ws for ASR. Fail-closed design — no
  ADMIN_DEMO_TOKEN means no mutation endpoints (admin-auth.ts:22).

  The concierge loop (concierge.ts) is the heart: model → (tool calls → tool
  results → model)* → text, capped at MAX_HOPS=6. System prompt
  (data/prompts/concierge.system.md) makes deterministic tool-grounding the 
  "first and highest law" — the model may never invent a gate/section/route;
  every fact must come from a tool result. Tools (tools.ts) resolve fuzzy labels
  to real node IDs via the graph loader and route with A*; node IDs never leak
  to the model. User input is wrapped in <fan_message> sentinels
  (concierge.ts:52,71) as prompt-injection defense.

  A* router (astar.ts): clean min-heap implementation, admissible haversine/1.4
  m·s⁻¹ heuristic, mode-aware weighting that only adds cost (step-free = +100k
  for stairs, sensory-safe = softer, low-crowd = quadratic density penalty).
  Accessibility is a weight, never a filter → a route always exists. Supports a
  blockedNode callback so operational advisories re-route around closed nodes.

  Crowd simulator (simulator.ts): honestly labelled source: "sim", derived from
  the real graph (zones = level × kind), driven by a matchday keyframe curve
  with T+15/T+30 projections. Admin can pin the clock (demo) or override a zone.

  LLM seam (provider.ts / qwen.ts): single interface, Qwen-only impl.
  Token-bucket concurrency guard (rate-limit.ts) with an abort-aware wait queue
  so disconnected clients don't pile up. ASR/TTS/vision all share the DashScope
  key server-side.

  ---
  Frontend

  Stack: React 18 + Vite 5 + Tailwind, React Router 6, Cesium/Resium (3D),
  Leaflet (2D), R3F/three (trophy hero), react-markdown. No global store — just
  two contexts (A11yContext, AlertContext); feature state is route-local. Shared
  Zod schemas validate every backend response client-side. Lazy-loaded routes +
  lazy 3D + deferred trophy + PWA (Workbox) keep the phone-conscious promise.

  Routes: / Landing (hero + 3D trophy + feature grid), /concierge (chat + 2D/3D
  map, 10 response languages, RTL, GPS opt-in, sign-reader + live captions),
  /navigate (tactical map, from/to/mode, floor tabs, forecast toggle,
  auto-replans on SSE alerts intersecting the route), /admin (Bearer-gated ops
  console: AI briefing, PA translator, guided-demo toggle, incident injection),
  * 404.

  3D map (StadiumMap3D.tsx, 1160 lines) is the centerpiece: two modes —
  photorealistic (Google 3D Tiles) and structure (MetLife's 8 real Mappedin
  levels extruded as a stacked cutaway). Find-my-seat search over 367 real bowl
  sections, concierge-driven section highlight, vertical-circulation columns
  (elevators/escalators/stairs/ramps) drawn through real per-floor stops,
  facility pins, indoor route that climbs the stack. Handset GPU profiling +
  requestRenderMode + render-loop paused when tab hidden. A ThreeDErrorBoundary
  permanently falls back to 2D on any Cesium crash.

  Realtime: AlertContext opens an EventSource to /api/alerts/stream with
  exponential-backoff reconnect; in production it errors out if VITE_API_BASE is
  unset (avoids Firebase's SPA-rewrite-returns-HTML trap). useLiveCaptions
  captures mic → downsamples to 16 kHz PCM16 → WS to backend ASR bridge, with an
  attempt-counter pattern that prevents stale callbacks.

  App Service the front-end appends the real client IP, so exposure depends on

  Real correctness bugs:

  3. occupancy_pct is mathematically mislabeled (admin.ts:212-218) — it's
  sum(densities) / count(zones) × 100 = "average zone density %", not stadium

  Cosmetic for a demo, but factually wrong.

  store); image_b64 accepted by ChatRequestSchema (up to 1.8 MB) but ignored by
